The Old Way

One Big Form for Everything

Django's default approach: one ModelForm with all fields, rendered on a separate page. You leave the list, navigate to an edit page, change what you need, submit, and go back. It works, but for quick edits — fixing a typo in a title, updating a year — it's heavy. You lose context, you wait for page loads, and the form shows fields you didn't want to touch.

Click "Edit" → Leave the list page → Full page with ALL fields → Change one field → Submit ALL fields → Redirect back to list 
# The classic approach: one ModelForm, all fields
class BookForm(forms.ModelForm):
    class Meta:
        model = Book
        fields = ["title", "author", "year"]

# One view for the whole thing
class BookUpdateView(UpdateView):
    model = Book
    form_class = BookForm
    template_name = "books/book_form.html"

The New Way

One Form Per Field + HTMx Inline Editing

Instead of one big form, you create one tiny form per field. Each is a ModelForm with exactly one field. When you click "Edit", HTMx fires a GET request that returns just the <form> fragment for that field. It swaps into the page without a full reload. You edit, save — the server validates and returns the updated value. The page never changes. You stay in context.

Click "Edit" → hx-get: fetch form fragment → Server returns <form> HTML → HTMx swaps it inline → Edit & Save → hx-post: submit one field → Validate, save, return display fragment 
# One form per field — a tiny ModelForm with exactly one field
class SingleFieldFormMixin:
    fieldname: str
    def get_field(self):
        return self[self.fieldname]   # the BoundField, for isolated rendering

class BookTitleForm(SingleFieldFormMixin, forms.ModelForm):
    fieldname = "title"
    class Meta:
        model = Book
        fields = ["title"]   # just this one field

# View: same URL returns a fragment for HTMx or a full page for browsers
def book_field_form(request, pk, fieldname):
    book = get_object_or_404(Book, pk=pk)
    form = get_forms(obj=book)[fieldname]
    template = "_field_form.html" if is_htmx(request) else "field_form.html"
    return render(request, template, {"object": book, "form": form})

Plugin Fields

New Fields Without Migrations

Regular model fields need a migration every time you add one. But what if you need per-tenant custom fields, or you just want to add a "notes" field quickly? Store values in a JSONField instead of a column. The JSONBackedMixin reads and writes to instance.misc[fieldname]. Adding a new editable field to the UI is one Python class — no makemigrations, no schema change. 

# The model has a JSONField for schemaless storage
class Book(models.Model):
    title = models.CharField(max_length=255)        # column
    author = models.CharField(max_length=255)       # column
    year = models.PositiveSmallIntegerField()        # column
    misc = models.JSONField(default=dict, blank=True)  # plugin fields live here

# Mixin: read/write from instance.misc instead of a column
class JSONBackedMixin:
    json_backed = True
    def __init__(self, instance=None, **kwargs):
        value = instance.misc.get(self.fieldname, "")
        kwargs.setdefault("initial", {})[self.fieldname] = value
        super().__init__(**kwargs)
    def save(self, **_):
        self.instance.misc[self.fieldname] = self.cleaned_data[self.fieldname]
        self.instance.save()

# Adding a new field = adding one class. No migration.
class TriviaForm(JSONBackedMixin, SingleFieldFormMixin, forms.Form):
    fieldname = "trivia"
    trivia = forms.CharField(widget=forms.Textarea, required=False)

Registry

Auto-Discovery via __subclasses__()

Every SingleFieldFormMixin subclass is automatically discoverable. The view calls SingleFieldFormMixin.__subclasses__() to find all registered field forms. No manual list to maintain. Add a new form class, it appears in the UI. 

# The view walks every subclass — no hardcoded list
def get_forms(data=None, obj=None):
    forms = {}
    for Form in SingleFieldFormMixin.__subclasses__():
        kwargs = {"instance": obj}
        if Form.fieldname in data:
            kwargs["data"] = data
        forms[Form.fieldname] = Form(**kwargs)
    return forms

# For deeper hierarchies, use __init_subclass__ instead:
class BetterSingleFieldForm:
    registry: dict = {}
    def __init_subclass__(cls, **kwargs):
        super().__init_subclass__(**kwargs)
        if getattr(cls, "fieldname", None):
            cls.registry[cls.fieldname] = cls
