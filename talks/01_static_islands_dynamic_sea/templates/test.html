Mantle Experiment

Talk: "Static Islands in a Dynamic Sea" by Carlton Gibson.
Idea: Don't make Django typed. Build typed attrs classes on top of the ORM. Mantle reads those classes and auto-generates optimized queries. No manual only(), select_related(), or serializers.

Test 1 — Shape as serializer

Normally you'd write a BookmarkSerializer with a class Meta, field lists, and validation methods. With Mantle, you just declare a shape: BookmarkShape(url, title, comment). That's the entire serializer.

Look for: The response only has url, title, comment. No id, no favourite, no created_at, no user — Mantle only fetches and returns what the shape declares.
GET /api/bookmarks/

Test 2 — N+1 prevention (the big one)

The classic Django trap: loop over bookmarks, access b.user.username each time, and Django fires a separate SQL query per bookmark. With 1000 bookmarks that's 1001 queries. Mantle sees user: UserData nested in your shape and auto-generates prefetch_related. Same data, fraction of the queries.

Look for: Both sides return identical data. But the left side (naive) shows queries_executed: 7 (1 + 6 per bookmark). The right side (Mantle) does it in 2.

Test 3 — Field flattening with @overrides

Sometimes you want "username": "alice" at the top level instead of "user": {"username": "alice"}. The @overrides decorator annotates F("user__username") and Mantle generates a single JOIN — no extra query, no Python-side reshaping.

Look for: username is a top-level field next to url and title. No nested user object.
GET /api/bookmarks/flat/

Test 4 — Create with validation

Mantle's compose_validators(unique_field("url")) checks constraints before inserting. Submit once — it works. Submit again with the same URL — Mantle rejects it.

Look for: First submit returns "created". Second submit with the same URL returns "error": "Bookmark with url='...' already exists".
POST /api/bookmarks/validated/

Test 5 — Update (shape fields only)

mantle.update() only touches fields in the shape. The bookmark's user stays unchanged because BookmarkData doesn't include it. Validation still runs — try changing the URL to one that already exists.

Look for: The title changes. Then try setting the URL to https://docs.djangoproject.com/ (already taken) — it should fail validation.
PUT /api/bookmarks/{id}/update/
