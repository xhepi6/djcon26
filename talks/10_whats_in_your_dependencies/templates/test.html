Talk 10 — What's in Your Dependencies?

Supply chain security for Django projects: scan, evaluate, and harden your dependencies.

i

What you're looking at

A live demo of the supply chain defense toolkit from the talk. Every pip install runs untrusted code with your privileges — no sandboxing. Use the panels below to scan your environment, evaluate a package before installing it, and see what hash-pinned requirements look like.
1

Scan for vulnerabilities (pip-audit)

pip-audit checks every installed package against the OSV (Open Source Vulnerabilities) database. Run this in CI on every PR so you catch new CVEs as they're published — not after you've been breached. Also available as a GitHub Action: pypa/gh-action-pip-audit.
2

Evaluate a package before installing

Before adding any new dependency: query the PyPI API for health signals. A brand-new package with no history, no source repo, and no license is a red flag. Also check deps.dev and Socket.dev for deeper analysis.
3

Hash-pinned requirements

A version pin (django==5.1.3) tells pip which version to install. A hash pin tells pip exactly which file to accept. If the file on PyPI changes — even at the same version — installation fails. This protects against compromised maintainer accounts and mirror tampering.
4

Real incidents from the talk

These attacks are not theoretical. They happened on PyPI and affected real packages with real download counts. Understanding them shapes how you defend.
jeIlyfish / python3-dateutil
Typosquatting
jeIlyfish (capital I instead of L) impersonated jellyfish. Paired with python3-dateutil. Both stole SSH and GPG keys.
On PyPI undetected for ~1 year (2019)
Colorama typosquats
Typosquatting + malware
500+ fake variations of colorama deployed zgRAT malware via a fake PyPI mirror domain.
2024 — discovered via domain analysis
deep-translator phishing
Phishing maintainers
Attackers impersonated PyPI with a fake "mandatory validation" email. Maintainers of deep-translator, exotel, and spam had credentials stolen. Malicious versions published under their accounts.
August 2022
Codecov Bash Uploader
CI/CD pipeline attack
Attackers used leaked Docker credentials to modify Codecov's Bash Uploader. For ~2 months it exfiltrated environment variables from 23,000+ customers (Twilio, HashiCorp, etc.).
2021 — 2 months undetected
Ultralytics YOLO
GitHub Actions injection
Attackers exploited pull_request_target + template injection via a branch name to inject a crypto miner into PyPI versions 8.3.41 and 8.3.42.
~60M downloads, 30K+ stars (2024)
ctx package
Account hijacking
Dormant for ~8 years. Attacker bought the expired domain of the maintainer's email, then used PyPI's password reset to take over the account. Published versions that stole AWS environment variables.
~27,000 malicious downloads (2022)
Dependency confusion
Name collision attack
Alex Birsan published public packages with the same names as internal private packages at Apple, Microsoft, PayPal, Tesla. Package managers installed the public one because it had a higher version number.
35+ companies breached, $130K+ bounties (2021)
AI-hallucinated packages
Emerging threat
LLMs sometimes suggest package names that don't exist. An attacker can register that name on PyPI. Developers copy-paste the install command and run malicious code without knowing.
Active — growing concern as AI code gen adoption increases
