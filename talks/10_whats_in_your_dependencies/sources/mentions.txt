## REAL SUPPLY CHAIN INCIDENTS

### Typosquatting attacks on PyPI

- python3-dateutil + jeIlyfish (capital I instead of lowercase L) — December 2019
  Stole SSH and GPG keys. jeIlyfish was on PyPI for ~1 year before detection.
  Discovered by German developer Lukas Martini.
  https://www.sysdig.com/blog/malicious-python-libraries-jeilyfish-dateutil
  https://lwn.net/Articles/806266/
  https://github.com/dateutil/dateutil/issues/984

- Colorama typosquatting campaigns — March 2024
  500+ typosquatted variations of colorama deployed, linked to zgRAT malware.
  A fake Python mirror (files[.]pypihosted[.]org) hosted tampered colorama.
  https://www.imperva.com/blog/pythons-colorama-typosquatting-meets-fade-stealer-malware/
  https://checkmarx.com/zero-post/python-pypi-supply-chain-attack-colorama/
  https://blog.pypi.org/posts/2024-04-10-domain-abuse/

- Large-scale typosquatting campaign — late 2023
  Hundreds of malicious packages mimicking popular libraries.
  https://www.techtarget.com/searchsecurity/news/366577455/Typosquatting-campaign-malicious-packages-slam-PyPi
  https://blog.checkpoint.com/securing-the-cloud/pypi-inundated-by-malicious-typosquatting-campaign/


### Phishing attacks on PyPI maintainers — August 2022

- First known phishing attack directly targeting PyPI.
  Emails impersonated PyPI, warned of mandatory "validation process."
  Phishing site at pypi-mirror.org stole credentials.
  Compromised packages included "deep-translator", "exotel", "spam".
  Several hundred typosquats also removed.
  https://thehackernews.com/2022/08/pypi-repository-warns-python-project.html
  https://www.bleepingcomputer.com/news/security/pypi-urges-users-to-reset-credentials-after-new-phishing-attacks/
  https://www.helpnetsecurity.com/2022/08/25/pypi-phishing/


### CI/CD pipeline attacks

- Codecov Bash Uploader compromise — January-April 2021
  Attackers modified Codecov's Bash Uploader script via leaked Docker credentials.
  Exfiltrated environment variables (API keys, tokens) for ~2 months.
  Affected 23,000+ customers including Twilio, HashiCorp, Rapid7, Confluent.
  https://about.codecov.io/security-update/
  https://about.codecov.io/apr-2021-post-mortem/
  https://blog.gitguardian.com/codecov-supply-chain-breach/

- Ultralytics YOLO supply chain attack — December 4-7, 2024
  Exploited GitHub Actions pull_request_target trigger + template injection via branch name.
  Injected XMRig crypto miner into versions 8.3.41 and 8.3.42 on PyPI.
  30K+ GitHub stars, ~60M PyPI downloads. Compromised versions available 12+ hours.
  https://blog.pypi.org/posts/2024-12-11-ultralytics-attack-analysis/
  https://www.reversinglabs.com/blog/compromised-ultralytics-pypi-package-delivers-crypto-coinminer
  https://snyk.io/blog/ultralytics-ai-pwn-request-supply-chain-attack/
  https://www.wiz.io/blog/ultralytics-ai-library-hacked-via-github-for-cryptomining


### ctx package hijack — May 2022

- Dormant for ~8 years. Attacker purchased the expired domain of the maintainer's email,
  used PyPI password reset to take over the account.
  Malicious versions stole environment variables (including AWS keys).
  ~27,000 malicious copies downloaded.
  Simultaneous attack on PHP's phpass library by same actor.
  https://thehackernews.com/2022/05/pypi-package-ctx-and-php-library-phpass.html
  https://www.bleepingcomputer.com/news/security/popular-python-and-php-libraries-hijacked-to-steal-aws-keys/
  https://orca.security/resources/blog/python-supply-chain-attack-ctx-phpass/


### Crypto miners injected into packages

- Ultralytics (December 2024): see CI/CD section above
- modularseven, driftme, catme — December 2023
  Three packages by author "sastra" deployed CoinMiner on Linux.
  431 downloads before removal.
  https://www.fortinet.com/blog/threat-research/malicious-pypi-packages-deploy-coinminer-on-linux-devices
  https://thehackernews.com/2024/01/beware-3-malicious-pypi-packages-found.html


### event-stream npm incident — November 2018 (comparison reference)

- Maintainer (Dominic Tarr) handed over access to "right9ctrl" who added malicious
  flatmap-stream dependency targeting BitPay Copay wallet.
  Stole cryptocurrency from wallets with >100 BTC.
  Ran undetected for 2.5 months. Affected ~800K projects.
  https://blog.npmjs.org/post/180565383195/details-about-the-event-stream-incident
  https://medium.com/intrinsic-blog/compromised-npm-package-event-stream-d47d08605502


### Dependency confusion — February 2021

- Alex Birsan breached 35+ companies (Apple, Microsoft, PayPal, Tesla, Uber, etc.)
  by publishing public packages with same names as internal private packages.
  Package managers installed public version due to higher version number.
  $130K+ in bug bounties. Foundational supply chain research.
  https://medium.com/@alex.birsan/dependency-confusion-4a5d60fec610
  https://blog.gitguardian.com/dependency-confusion-attacks/

## DEFENSE TOOLS & TECHNIQUES

### pip-audit (PyPA / Trail of Bits + Google)
- Scans Python environments for packages with known vulnerabilities.
- Uses PyPI JSON API + OSV database.
- Supports automatic fixes, multiple output formats, GitHub Action integration.
- GitHub: https://github.com/pypa/pip-audit
- PyPI: https://pypi.org/project/pip-audit/
- GitHub Action: https://github.com/pypa/gh-action-pip-audit

### Safety CLI (Safety Cybersecurity, formerly PyUp)
- Vulnerability scanner using Safety DB.
- Commands: safety scan, safety system-scan, safety scan --apply-fixes.
- GitHub: https://github.com/pyupio/safety
- PyPI: https://pypi.org/project/safety/
- Website: https://pyup.io/safety/
- Docs: https://docs.safetycli.com/safety-2

### OSV (Open Source Vulnerabilities) — Google
- Aggregates vulnerability databases (GitHub Security Advisories, PyPA, RustSec, etc.).
- Covers PyPI, npm, Go, Rust, and more.
- Free API for querying by commit hash or package version.
- Website: https://osv.dev/
- GitHub: https://github.com/google/osv.dev
- Scanner: https://github.com/google/osv-scanner
- Docs: https://google.github.io/osv.dev/

### Snyk
- Developer security platform for vulnerability scanning in dependencies.
- Vulnerability database: https://security.snyk.io
- Python security: https://snyk.io/platform/snyk-python-security/
- Open Source scanning: https://snyk.io/product/open-source-security-management/
- GitHub: https://github.com/snyk
- Note: Snyk Advisor (package health) sunset January 2026, replaced by Snyk Security DB.

### Socket.dev
- Detects supply chain attacks via package behavior analysis (not just CVEs).
- 70+ security red flags. Detects unstable ownership, out-of-order publishing.
- Free for open source repos. GitHub app for PR-level monitoring.
- Website: https://socket.dev/
- Blog: https://socket.dev/blog

### Sigstore — Package signing
- Keyless signing tied to OIDC identity (GitHub, Google accounts).
- Components: Cosign (signing), Fulcio (certificates), Rekor (transparency log).
- Python client: https://github.com/sigstore/sigstore-python
- PyPI: https://pypi.org/project/sigstore/
- Blog: https://blog.sigstore.dev/announcing-the-1-0-release-of-sigstore-python-4f5d718b468d/
- Python & Sigstore (Seth Larson): https://sethmlarson.dev/python-and-sigstore
- Official: https://www.sigstore.dev/ (implied from blog.sigstore.dev)

### uv (Astral) — Package manager with security features
- --require-hashes flag for hash-checking mode (SHA-256, SHA-384, SHA-512).
- Default "first-index" resolution prevents dependency confusion.
- uv lock / uv pip compile --generate-hashes for pinned lockfiles.
- Website: https://docs.astral.sh/uv/
- GitHub: https://github.com/astral-sh/uv
- Security overview: https://github.com/astral-sh/uv/security
- Security analysis: https://pydevtools.com/blog/astral-security-post-what-to-keep/

### pip hash checking (--require-hashes)
- All-or-nothing hash verification. SHA-256 recommended.
- Works with pip install, pip download, pip wheel.
- Docs: https://pip.pypa.io/en/stable/topics/secure-installs/
- pip hash command: https://pip.pypa.io/en/stable/cli/pip_hash/

### Trusted Publishers on PyPI (OIDC)
- Eliminates long-lived API tokens. Uses short-lived OIDC tokens.
- Supports GitHub Actions, GitLab, Google Cloud, ActiveState.
- Docs: https://docs.pypi.org/trusted-publishers/
- Internals: https://docs.pypi.org/trusted-publishers/internals/
- Announcement: https://blog.pypi.org/posts/2023-04-20-introducing-trusted-publishers/
- GitHub OIDC config: https://docs.github.com/actions/deployment/security-hardening-your-deployments/configuring-openid-connect-in-pypi
- PEP 807 (Index support for Trusted Publishing): https://peps.python.org/pep-0807/

### PEP 740 — Digital Attestations on PyPI
- Sigstore-based attestations for package provenance. GA November 2024.
- Automatic with Trusted Publishing + pypa/gh-action-pypi-publish v1.11.0+.
- 132,360+ packages with attestations as of March 2026.
- PEP: https://peps.python.org/pep-0740/
- Announcement: https://blog.pypi.org/posts/2024-11-14-pypi-now-supports-digital-attestations/
- Trail of Bits blog: https://blog.trailofbits.com/2024/11/14/attestations-a-new-generation-of-signatures-on-pypi/
- Adoption tracker: https://trailofbits.github.io/are-we-pep740-yet/

### PyPI malware checks
- Official scanning has faced false-positive challenges (~1/3 of popular packages trigger alerts).
- Quarantine system (2025): freeze suspected malware for investigation.
- 2,000+ malware reports processed in 2025, 66% handled within 4 hours.
- Mostly relies on volunteer security researchers + community reporting.
- GuardDog (Datadog): static analysis tool for PyPI/npm malware detection.
  https://github.com/DataDog/guarddog
  https://securitylabs.datadoghq.com/articles/guarddog-identify-malicious-pypi-packages/
- PyPI security page: https://pypi.org/security/

### npm sandboxing comparison
- npm runs install scripts (postinstall) with full system access by default.
- pnpm blocks risky post-install scripts by default.
- Deno provides sandboxed npm execution: https://deno.com/blog/deno-protects-npm-exploits
- Python/pip has the same problem: setup.py runs arbitrary code during install.
- virtualenv/venv provide NO sandboxing — just as dangerous as outside.
- Reference: https://www.codethink.co.uk/articles/2023/pypi-safety/

### Docker isolation for package installation
- Run pip install inside containers to limit blast radius.
- Run pip-audit in isolated Docker container before installing on dev machine.
- Multi-stage builds: remove pip/build tools from final image.
- Reference: https://bernat.tech/posts/securing-python-supply-chain/
- Reference: https://docs.bswen.com/blog/2026-02-21-pip-supply-chain-security/

### deps.dev (Google Open Source Insights)
- Tracks 5M+ packages across PyPI, npm, Maven, Go, Cargo.
- Transitive dependency graphs, advisory impact, OpenSSF Scorecard data.
- Free API and BigQuery dataset.
- Website: https://deps.dev/
- GitHub: https://github.com/google/deps.dev
- Docs: https://docs.deps.dev/
- Blog: https://blog.deps.dev/
- PyPI attestations support: https://blog.deps.dev/pypi-attestations/

### OpenSSF Scorecard
- Automated security health checks for open source projects (0-10 scores).
- Checks: dependency management, CI usage, security testing, code review, etc.
- Weekly scan of 1M most critical projects. Results in BigQuery.
- Website: https://scorecard.dev/
- GitHub: https://github.com/ossf/scorecard
- OpenSSF page: https://openssf.org/projects/scorecard/
- Checks documentation: https://github.com/ossf/scorecard/blob/main/docs/checks.md

### Libraries.io
- Monitors 2M+ packages from 36 package managers.
- Tracks releases, licenses, contributors, dependencies.
- Free search and API.
- Website: https://libraries.io/
- GitHub: https://github.com/librariesio
- API docs: https://libraries.io/api
